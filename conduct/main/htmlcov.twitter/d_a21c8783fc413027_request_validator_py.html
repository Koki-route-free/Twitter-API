41 def authenticate_client(self, request, *args, **kwargs): 

42 """Authenticate client through means outside the OAuth 2 spec. 

43 

44 Means of authentication is negotiated beforehand and may for example 

45 be `HTTP Basic Authentication Scheme`_ which utilizes the Authorization 

46 header. 

47 

48 Headers may be accesses through request.headers and parameters found in 

49 both body and query can be obtained by direct attribute access, i.e. 

50 request.client_id for client_id in the URL query. 

51  

52 The authentication process is required to contain the identification of 

53 the client (i.e. search the database based on the client_id). In case the 

54 client doesn't exist based on the received client_id, this method has to 

55 return False and the HTTP response created by the library will contain 

56 'invalid_client' message.  

57 

58 After the client identification succeeds, this method needs to set the 

59 client on the request, i.e. request.client = client. A client object's 

60 class must contain the 'client_id' attribute and the 'client_id' must have 

61 a value. 

62 

63 :param request: OAuthlib request. 

64 :type request: oauthlib.common.Request 

65 :rtype: True or False 

66 

67 Method is used by: 

68 - Authorization Code Grant 

69 - Resource Owner Password Credentials Grant (may be disabled)

366 def validate_bearer_token(self, token, scopes, request): 

367 """Ensure the Bearer token is valid and authorized access to scopes. 

368 

369 :param token: A string of random characters. 

370 :param scopes: A list of scopes associated with the protected resource. 

371 :param request: OAuthlib request. 

372 :type request: oauthlib.common.Request 

373 

374 A key to OAuth 2 security and restricting impact of leaked tokens is 

375 the short expiration time of tokens, *always ensure the token has not 

376 expired!*. 

377 

378 Two different approaches to scope validation: 

379 

380 1) all(scopes). The token must be authorized access to all scopes 

381 associated with the resource. For example, the 

382 token has access to ``read-only`` and ``images``, 

383 thus the client can view images but not upload new. 

384 Allows for fine grained access control through 

385 combining various scopes. 

386 

387 2) any(scopes). The token must be authorized access to one of the 

388 scopes associated with the resource. For example, 

389 token has access to ``read-only-images``. 

390 Allows for fine grained, although arguably less 

391 convenient, access control. 

392 

393 A powerful way to use scopes would mimic UNIX ACLs and see a scope 

394 as a group with certain privileges. For a restful API these might 

395 map to HTTP verbs instead of read, write and execute. 

396 

397 Note, the request.user attribute can be set to the resource owner 

398 associated with this token. Similarly the request.client and 

399 request.scopes attribute can be set to associated client object 

400 and authorized scopes. If you then use a decorator such as the 

401 one provided for django these attributes will be made available 

402 in all protected views as keyword arguments.
